Execution context in JavaScript

Execution context is an important concept in JavaScript that defines the environment in which code is executed. It includes variables, functions, and objects that are accessible during the execution of a particular piece of code. Understanding execution contexts is crucial for understanding how JavaScript code is executed.

There are three types of execution contexts in JavaScript:

1. Global Execution Context: This is the default or outermost execution context. It represents the global scope and is created when the JavaScript engine starts executing the script. All variables and functions declared outside of any function belong to the global execution context. There is only one global execution context per page.

2. Function Execution Context: A new function execution context is created whenever a function is invoked or called. Each function has its own execution context, which includes function-specific variables, parameters, and inner functions. Function execution contexts are created and destroyed as functions are invoked and completed.

3. Eval Execution Context: The eval function in JavaScript can be used to execute code dynamically. When eval is used, a new execution context called the eval execution context is created. It represents the scope of the evaluated code.
